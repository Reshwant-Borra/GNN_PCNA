## Split Goal

Create a leakage-resistant benchmark split for residue-level candidate cryptic-pocket prediction, prioritizing scientific trustworthiness over dataset size.

## Candidate Split S1: Audited CryptoBench Split

Use the official CryptoBench split as a starting point only after local files are available. Audit and modify only under a documented pre-result protocol if any Phase 2 leakage rule fails.

Required checks:

- No shared UniProt ID across train, validation, test, and held-out PCNA/control sets.
- No high-identity homologs across splits under the approved sequence clustering threshold.
- No apo/holo pairs, repeated apo structures, holo alternatives, ligand variants, or duplicate biological assemblies across splits.
- No PCNA, PCNA homolog, or PCNA-like sliding-clamp system in train/validation if PCNA final-claim evaluation is planned.
- No benchmark-provided split accepted without hashes and leakage audit.

Strength: preserves published benchmark comparability if it passes.

Risk: official split may not satisfy Phase 2 grouping; local files are not yet present.

## Candidate Split S2: Strict Phase 2 De Novo Split

Build a fresh split from CryptoBench records after inventory. This is preferred if any official split audit fails.

Proposed grouping keys:

- `system_group_id`: UniProt accession plus protein family annotation.
- `sequence_cluster_id`: generated by approved tool and threshold.
- `apo_group_id`: apo PDB ID, chain, biological assembly, and mapped UniProt.
- `holo_group_id`: all paired holo PDB IDs/chains for the apo group.
- `ligand_variant_group_id`: ligand ID and chemically similar variants where known.
- `structure_duplicate_group_id`: duplicate PDB IDs, chains, assemblies, redundant homomers, and identical coordinate hashes.
- `pcna_holdout_flag`: PCNA/P12004, PCNA homolog, sliding-clamp structural homolog, 8GLA/AOH1996-related, or unresolved PCNA-like status.

Proposed assignment:

- Train: non-PCNA, non-heldout groups only.
- Validation: non-PCNA groups disjoint by all grouping keys; used only for model selection.
- Test: non-PCNA groups disjoint by all grouping keys; used once after model, thresholds, baselines, and report plan are frozen.
- Held-out PCNA final-claim set: all PCNA/P12004 structures and close homologs, if any, excluded from model selection.
- Positive controls: 8GLA/ZQZ and other declared controls only if isolated and not used for tuning.
- Excluded/quarantined: unresolved chain identity, unresolved ligand identity, unclear license, missing files, bad residue mapping, or leakage conflicts.

Strength: strongest governance compliance.

Risk: reduces benchmark comparability and may reduce dataset size.

## Candidate Split S3: Evaluation-Only CryptoBench

Use CryptoBench only as a frozen external evaluation benchmark after training on a separate governed source.

Required condition: no overlap between training source and CryptoBench by UniProt, homolog cluster, apo/holo system, ligand variant, or structural similarity.

Strength: external benchmark framing.

Risk: impossible if auxiliary training sources overlap broadly with PDB-derived CryptoBench records.

## Required Split Artifacts Before Freeze

- `data/splits/phase2_split_<hash>.json`
- `data/splits/phase2_split_<hash>.sha256.txt`
- Sequence cluster table with tool, version, threshold, command, input hashes, and output hash.
- Apo/holo grouping table.
- Duplicate PDB/chain/assembly table.
- Ligand variant grouping table.
- PCNA/homolog isolation statement.
- Structural similarity review notes for boundary cases.
- Human review record in `reports/phase2/human_review_log.md`.

## Current Blockers

- No local CryptoBench dataset files or hashes.
- Approved sequence clustering tool and threshold are not frozen.
- PCNA/homolog screen has not been run.
- Apo/holo and duplicate grouping tables do not exist.
- Human split-freeze reviewer/decision is missing.

## Recommendation

Prefer S2 if the goal is a canonical Phase 2 foundation. S1 is acceptable only if it passes every Phase 2 leakage check. S3 is a fallback if CryptoBench is better preserved as external evaluation than as a training benchmark.

Current readiness: not ready for split freeze.
